项目名称：微信小程序记账与AI理财助手（应用名：昕辰记账）

【开发状态】第一阶段 ✅ 已完成（2026-07-01）
  已完成：项目骨架、Storage服务层、手动记账MVP、分类管理、账户管理、首页概览、设置页框架
  进行中：无
  待开发：P2 微信支付消息解析、P3 统计图表、P4 预算导入导出、P5 AI分析、P6 打磨上线

项目背景与目标：解决个人日常消费记录繁琐、经常忘记记账的问题。利用微信小程序接收用户转发的微信支付通知消息，自动解析金额和商户信息实现快速记账，同时保留手动记账作为补充。引入 AI 智能分析模块，基于收支数据自动生成消费画像和钱包管理建议。所有记账数据 100% 保存在本地设备（wx.Storage），无需网络即可使用核心记账功能；AI 分析支持纯本地规则引擎离线运行，也支持用户自愿配置 API Key 调用云端大模型（仅上传脱敏聚合数据），保障个人隐私安全。

1. 微信支付消息解析模块（核心模块）：

    1.1 原理说明：用户长按微信聊天中的「微信支付」公众号支付通知消息 → 点击「更多打开方式」→ 选择「昕辰记账」小程序。小程序在 App.onLaunch / App.onShow 中通过 referrerInfo.extraData 拿到消息文本内容，进行解析；

    1.2 小程序启动时检查场景参数，识别本次打开是否为转发消息进入；若为普通启动则进入首页，若携带微信支付消息则立即触发解析流程；

    1.3 解析通知文本内容，通过正则表达式提取交易金额（如从"微信支付-消费 ¥38.00"中提取 38.00）、交易方向（收入/支出，根据关键词"消费""支付""收款""转入""退款""红包"等判断）。特别注意退款场景：退款通知通常包含"退款""到账""退回"等关键词，应标记为收入而非支出；

    1.4 根据通知文本中的商户名或交易描述，匹配内置的关键词映射表，自动归入相应分类（如含"麦当劳""食堂""美团"归为"餐饮"，含"滴滴""地铁"归为"交通"）；

    1.5 解析成功后弹出确认半弹层（而非静默入库），展示解析结果（金额、分类、商户名），用户可修改后点确认保存，或点取消丢弃。原因：消息转发是用户主动触发的，给确认机会比静默入库体验更好；

    1.6 确认保存后将记录存入本地 Storage，字段包括：金额（Number）、类型（收入/支出）、分类（字符串）、商户/描述（原始消息文本截取）、交易时间（时间戳）、数据来源（标记为"wechat"）；

    1.7 去重机制：若 10 秒内已存在相同金额且来源为"wechat"的记录，视为重复转发，不再重复入库，toast 提示"已存在，跳过"；

    1.8 对于无法解析的消息（正则未匹配到金额），将原始文本完整记录在日志中，并 toast 提示"未识别到金额，请在日志中查看"，用户可在设置页面查看和手动补录；

    1.9 使用指引：首次使用此功能时，弹窗图文引导用户如何转发微信支付消息到小程序（截图+箭头指示：长按支付消息 → 更多打开方式 → 昕辰记账）。

11. AI 智能分析与钱包管理建议模块：

    11.1 周期账单聚合：按月、季度自动汇总用户的收支总额、各分类占比、环比增减幅度、日均支出等核心指标，生成结构化的摘要数据（如"本月总收入5000元，总支出4200元，餐饮占比40%，交通占比10%"），作为 AI 分析的上下文；

    11.2 消费习惯洞察：自动识别高频小额消费（如一周内奶茶/咖啡类消费超过 3 次）、连续超支分类（某分类支出已超过上月同期的 120%）、大额异常支出（单笔超过近 30 天平均单笔支出的 3 倍），并标记为"需注意"项；

    11.3 智能建议生成（报告模式）：用户点击"生成我的消费报告"按钮，系统基于近期账单数据生成一份自然语言的钱包管理建议。例如："本月餐饮支出占比达 42%（¥1,260），其中外卖订单 28 笔，建议在接下来两周减少外卖频次，预计可节省约 ¥300"；"本月结余率仅为 5%，建议设定每月工资到账后立即转入储蓄账户 20% 的计划"；

    11.4 AI 对话助手（交互模式）：在 App 内设立"AI 助手"页面，除了生成固定报告外，用户可在输入框中自由提问，如"我这个月哪里花多了？""给我定一个下个月的存钱计划""我每天少喝一杯奶茶一年能省多少？"，系统结合近期账单数据构造 Prompt 调用 AI 作答；

    11.5 本地规则引擎兜底：若用户未配置云端 AI API Key，则使用内置的 JavaScript 规则引擎（if-else 逻辑）生成基础建议。例如检测到餐饮超支即提示"餐饮超支警告，建议控制饮食开支"，检测到连续三天有外卖订单提示"注意休息和健康，适当自己做饭"；保证无网络环境下也有基本智能反馈；

    11.6 数据脱敏与隐私保护：若调用云端 AI，系统仅将分类汇总数据（如"餐饮:1260元, 交通:200元"）和匿名化的问题发送至 API，绝不发送具体的商户名称、交易时间、真实姓名等敏感原始明细；在首次发送时弹窗告知用户"本次将发送脱敏数据至AI模型，是否继续"；

    11.7 AI 建议历史：将所有生成的 AI 建议和分析报告以对话/报告形式保存在本地 Storage 的 ai_reports 中，用户可随时在"历史建议"列表中回溯查看，方便对比不同月份的改进情况；

    11.8 AI 账单标签：AI 分析每笔消费后，可为消费记录自动打标签（如"必要支出""可选支出""冲动消费""固定开销"），辅助用户识别可削减开支。

二、性能与安全需求
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

性能需求：
    - 小程序启动冷启动 < 2 秒，热启动 < 1 秒；
    - 消息解析在 500ms 内完成（含正则匹配+关键词分类）；
    - 本地 Storage 读写极快，1000 条记录下首页加载 < 300ms，统计图表计算 < 1 秒；
    - 代码包大小 < 2MB（微信小程序主包限制），分包后总大小 < 20MB；
    - ECharts 约 900KB 放入分包，避免主包过大；
    - CSV 导出 5000 条记录需在 5 秒内完成；
    - AI 云端接口调用时显示 loading 占位动画，响应超时 15 秒给予提示并可重试；
    - 本地规则引擎建议生成需在 0.5 秒内完成。

安全需求：
    - 所有记账数据 100% 存储在本地设备 wx.Storage 中，核心记账功能零网络通信；
    - 若用户主动填入 API Key 启用云端 AI，需明确弹窗告知用户数据将发送至第三方大模型服务商，且仅发送脱敏聚合数据（不包含商户名、交易时间、个人身份信息等）；
    - API Key 加密后存储在本地 Storage，绝不硬编码在代码中；
    - 遵循最小权限原则，小程序不申请任何多余 scope（如地理位置、通讯录等）；
    - 导出 CSV 时不包含任何设备标识或用户身份信息；
    - 消息转发解析仅处理用户主动转发的内容，不读取用户聊天记录。

三、技术架构与项目要求
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. 开发语言与框架：
    1.1 开发语言：JavaScript（ES6+）
    1.2 框架：微信小程序原生框架（WXML + WXSS + JS）
    1.3 基础库版本要求：最低 2.20.0（支持 shareDataMessage 等消息转发 API）
    1.4 开发工具：微信开发者工具（官方 IDE）+ VS Code 辅助编辑
    1.5 注意：不使用 TypeScript、不使用 Taro/uni-app 等跨端框架，保持技术栈最简

2. 界面框架：
    2.1 使用微信小程序原生 WXML + WXSS 编写界面
    2.2 WXSS 设计基准：iPhone 6 屏幕宽度 375pt = 750rpx（1pt = 2rpx）
    2.3 保留马卡龙萌系视觉风格（见第五节视觉规范），WXSS 自定义属性（CSS 变量）实现全局主题色
    2.4 仅实现浅色主题

3. 架构模式：
    3.1 采用简单的"页面-服务-工具"三层分离：
        - View 层：WXML 模板 + WXSS 样式
        - Logic 层：Page/Component 的 JS，调用 services 层
        - Data 层：services/ 封装 wx.Storage 读写操作
    3.2 全局状态：使用 App.globalData 存储当前用户配置，启动时从 Storage 加载到内存
    3.3 不使用 mobx / redux 等状态管理库

4. 本地数据存储（wx.Storage）：

    使用 wx.setStorageSync / wx.getStorageSync 系列 API。数据以 JSON 序列化存储：

    Storage Key              | 存储内容             | 数据结构
    ─────────────────────────────────────────────────────
    transactions             | 所有交易记录数组       | Array<Transaction>
    categories               | 分类列表              | Array<Category>
    accounts                 | 账户列表              | Array<Account>
    rules                    | 关键词映射规则列表      | Array<Rule>
    parse_logs               | 消息解析日志数组       | Array<ParseLog>
    budgets                  | 预算设置              | Array<Budget>
    ai_reports               | AI 建议历史           | Array<AiReport>
    app_config               | 应用配置              | Object

    各数据结构定义：

    Transaction {
      id: String,
      amount: Number,
      type: String,          // "income" | "expense"
      categoryId: String,
      categoryName: String,
      accountId: String,

7. 微信小程序特有约束
    - 自定义 tabBar 在 app.json 中配置 "tabBar": {"custom": true}
    - AI 云端 API 域名需在「微信公众平台 → 开发管理 → 服务器域名」配置白名单（如 api.deepseek.com）
    - 页面层级最多 10 层（wx.navigateTo 最多 10 次）
    - WXSS 不支持 backdrop-filter、position:fixed 在 scroll-view 内受限
    - 主包 2MB 上限，总包（含分包）20MB 上限
    - wx.Storage 单个 key 上限 1MB，总上限 10MB
    - 清除小程序数据会导致所有本地数据丢失 → 提醒用户定期导出 CSV
    - 微信小程序不允许使用 SSE（Server-Sent Events），云端 AI 需使用普通 HTTP 请求或 WebSocket 实现流式输出；如流式实现困难，可先用非流式（一次性返回完整响应）
